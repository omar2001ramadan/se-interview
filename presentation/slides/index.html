Graph Decision Loop

Inside the runtime architecture, the LangGraph loop keeps calling the model until it stops requesting tools.

Decision Flow
LLM Call

The first model call decides whether a tool is needed.

Tool Node

If a tool is called, its JSON output is written back into the conversation.

End

The loop ends only when the model stops emitting tool calls.

LangGraph

Tool Contracts

The assistant only needs three behaviors: planning, fresh facts, and an honest answer when the request requires a transaction.

Trip Planning

“What should I do in Chicago?”

Use search_attractions to gather organized destination evidence.

Current Info

“What is the weather in Lisbon?”

Use search_web when the answer depends on changing information.

Unsupported Action

“Book me a hotel in Rome.”

Do not pretend to transact. State the limit clearly and offer planning help instead.

Debugging Example

The Tokyo miss shows why traces matter: the issue was not “the answer looked bad,” but “the system made its decisions in the wrong order.”

Prompt

Mixed Intent

The request asks for nightlife planning and includes a freshness cue: “this weekend.”

Trace

Wrong First Tool

The first tool call was `search_web`, not the planning tool.

Diagnosis

Policy Gap

The prompt policy did not say how to prioritize mixed planning + freshness requests.

Fix

Route Planning First

Prefer destination planning first, then enrich with fresh information only when needed.

Why This Matters

Tracing turns a vague bad result into a precise system behavior that can be improved directly.

Operational Tradeoffs

Once the system is running as a service, the main operating concerns are latency, cost, and reliability.

Latency
  • Cache repeated planning and lookup results.
  • Keep the graph loop short and predictable.
  • Add timeouts around external calls.
Cost
  • Keep evaluation off the request path.
  • Sample traces in production when needed.
  • Use the smallest model that preserves routing quality.
Reliability
  • Fail honestly on unsupported transactions.
  • Degrade gracefully when retrieval fails.
  • Use traces to tighten policies over time.
